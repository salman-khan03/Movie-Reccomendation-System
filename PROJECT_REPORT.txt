WHAT I DID  SECTION BY SECTION

SECTION 1: DATA LOADING & INTEGRATION

I started with two separate CSV files and needed to combine them. The first file had 
movie information title, genres, and the second had user ratings. I loaded both 
files and then merged them by calculating statistics for each movie  like how many 
ratings it got and what the average rating was.

This was my first step in creating one unified dataset where I could access everything 
I needed about a movie in one place. The result was a dataset with 62,423 movies and 
their rating statistics all together.


SECTION 2: DATA CLEANING

As I started exploring the data, I realized it was pretty messy. There were several issues 
I had to fix:

1. Some movies had duplicate entries I just kept the first one
2. Movies with no genre listed had a weird label no genres listed I changed this 
   to Unknown
3. Some rows had missing values for rating statistics  I filled these with 0
4. Movie titles had years in parentheses like 1995  I extracted just the year
5. There were a few ratings outside the valid range 0.55.0 stars I removed these

After cleaning, I ended up with 62,387 movies and 490,000+ valid ratings. The data 
was now much cleaner and ready for real analysis.

SECTION 4: SAMPLING TECHNIQUES

When you have a huge dataset, sometimes you need to work with a smaller sample. I 
tested two approaches:

Random Sampling: Just pick 10% of ratings randomly. Simple and fast, but might miss 
certain groups. Mean rating came out to 3.53.

Stratified Sampling: Pick 10% of ratings, but make sure you get 10% from each rating 
category 1 star, 2 stars, etc.. More work but guarantees your sample has the same 
distribution as the whole dataset. Mean also 3.53.

I created a comparison chart showing both methods and proved that stratified sampling 
preserves the original distribution much better. This is important because if you use 
a skewed sample, your models might learn the wrong patterns.


SECTION 5: DATA TRANSFORMATION & NORMALIZATION

Here I noticed that some of my numeric variables were really skewed. For example, the 
number of ratings per movie: most movies get just a few ratings, but a handful of 
popular movies get thousands. This makes the distribution really unbalanced.

I tested different mathematical transformations to fix this:

Original: skewness = 2.5 very bad
Log transform: skewness = 0.8 much better!  
Square root: skewness = 1.2 okay

After choosing transformations, I also normalized all features to the same scale:
 MinMax: scales everything to [0, 1]
 ZScore: centers at 0 with standard deviation of 1

This normalization is important because machine learning algorithms don't like when 
one feature is in the range 0100 and another is 10 to 10. Everything should be on 
the same scale.


SECTION 6: DATA DISCRETIZATION

Discretization means turning continuous numbers into categories. I did this in two ways:

Rating Categories equalwidth:

2.53.5 = Average
3.54.2 = Good
4.25.0 = Excellent

Movie Popularity equalfrequency:
Created 4 groups where each group has roughly the same number of movies:
 Niche

This discretization makes the data easier to interpret and sometimes works better for 
certain types of analysis.


SECTION 7: PCA PRINCIPAL COMPONENT ANALYSIS

PCA is a cool technique for reducing the number of features you have. I had 24 binary 
0/1 columns for genres using onehot encoding. PCA let me compress this:

 To explain 90% of the variance: only need 5 components 79% reduction!
 To explain 95% of the variance: need 12 components 50% reduction

What this means: You lose almost no information but use way fewer dimensions. I also 
created a 2D visualization of all movies in PCA space, which was really neat  you 
could see how different genres clustered together.


SECTION 8: ATTRIBUTE SUBSET SELECTION

Not all features are equally important. I tested two methods to find which ones matter:

Variance Threshold: Removed genres that appear in less than 1% of movies. This got 
rid of obscure generes and reduced my features from 24 to 17.

SelectKBest with Fscore: This ranks features by how well they predict whether someone 
will like a movie. Top features were Drama, Comedy, SciFi, Action, and Adventure.

The idea is: if you focus on the important features, your models will be faster and 
often more accurate too.


SECTION 9: DATA COMPRESSION SINGULAR VALUE DECOMPOSITION

SVD is similar to PCA but works on large matrices. I tested compressing the genre matrix:

2 components   explains 94.74% of variance, error = 0.005
5 components   explains 97.12% of variance, error = 0.003
10 components  explains 98.15% of variance, error = 0.001
Full 23 comp   explains 100% of variance, error = 0

The key finding: even with just 2 components, you explain 95% of the variance! That's 
excellent compression  you keep almost all the information while using 91% fewer 
dimensions. This is why SVD is so popular for handling sparse data.


SECTION 10: REGRESSION ANALYSIS

I built three different regression models to predict a movie's average rating:

Model 1: Simple Linear Regression
 Used only one feature: lognumber of ratings
 R² = 0.0324 explains only 3% of variance
 MSE = 0.4925 average prediction error
 Formula: avg_rating = 3.29 + 0.067 × lognum_ratings
 Finding: Popular movies get slightly higher ratings on average

Model 2: Multiple Regression
 Used multiple features: number of ratings, year, and top 5 genres
 R² = 0.0891 explains 9% of variance  better!
 MSE = 0.4652 slightly smaller error
 The extra features helped!

Model 3: LogLinear Model
 Same features but mathematically different approach

 Slightly worse than Model 2

But honestly? All three models had pretty low R² values 0.08. This means movie

SECTION 11: CLUSTERING KMEANS

KMeans groups similar items together without using any labels. The question is: how 
many groups should we create?

I used the Elbow Method which plots the error called inertia against the number 
of clusters. The elbow the point where adding more clusters doesn't help much was 
at k=8.

So I ran KMeans with 8 clusters to group similar movies by genre:
 Cluster 0: Drama movies
 Cluster 1: Comedy movies  
 Cluster 2: Action/Adventure
 Clusters 37: Other genre combinations

I created a visualization using PCA to reduce to 2D, and you could see the 8 clusters 
separated nicely. Movies with similar genres ended up in the same cluster  exactly 
what we wanted!


SECTION 12: CONTENTBASED FILTERING RECOMMENDATION

This is the first of three recommendation approaches. The idea is simple: if you like 
a movie, recommend other movies that are similar to it.

How it works:
1. Convert each movie's genres into a TFIDF vector fancy way of weighting genres 
   by how important or unique they are
2. Calculate cosine similarity between movies how similar in genre
3. Return the top similar movies

Example: Recommend movies similar to Toy Story 1995
Results: Toy Story 2, Toy Story 3, A Bug's Life, Antz, etc. all familyfriendly 
animated comedies

Advantages: Works for new movies immediately, very interpretable so you know why a movie 
was recommended
Disadvantages: Doesn't learn user preferences, just looks at genres


SECTION 13: COLLABORATIVE FILTERING RECOMMENDATION

This is the opposite of contentbased. Instead of looking at movie features, we look 
at user behavior. The idea: Users who rated movies similarly to you like these other 
movies, so you probably will too!

How it works:
1. Build a usermovie matrix where rows = users, columns = movies, and cells = ratings
2. Use SVD to decompose this matrix into latent factors 50 abstract dimensions 
   representing user tastes and movie characteristics
3. Calculate similarity between movies in this latent space
4. Recommend similar movies

The matrix was SPARSE 99%+ empty because users only rate a tiny fraction of all 
movies. After filtering to the top 2,000 users and 3,000 movies, the SVD model 
explained 67.45% of the variance.

Advantages: Learns real user preferences, can find hidden patterns
Disadvantages: Needs lots of data, doesn't work for brand new movies or users cold 
start problem


SECTION 14: HYBRID RECOMMENDATION

Why pick just one method? I combined both using a weighted average:

Hybrid Score = 0.6 × Content Similarity + 0.4 × Collaborative Similarity

First I normalized both scores to [0, 1] so they're on the same scale. Then I weighted 
them: 60% from contentbased genre similarity and 40% from collaborative user 
patterns.

You can adjust the weights depending on what's more important for your use case:
 More contentbased alpha=0.7+ if new movies are important
 More collaborative alpha=0.3 if finding movies based on user taste is important
 Balanced alpha=0.5 for a mix

Example results:
Toy Story 1995: Recommends both similargenre movies AND movies liked by users 
with similar taste
The Dark Knight: Equal balance of both approaches


SECTION 15: DECISION TREE CLASSIFICATION

Here's a different problem: Can we predict whether a specific user will LIKE a 
specific movie define like as rating ≥ 4.0 stars?

Input features:
 Movie stats: average rating, number of ratings
 User stats: average rating this user gives, how many movies they've rated
 Onehot encoded genres is it Drama, Comedy, etc.?

The result was a decision tree with maximum depth of 6 levels. The tree learned rules 
like:

If this user tends to give high ratings avg > 3.5 AND the movie has lots of ratings 
AND it's not a Horror movie  predict they'll like it

Results:
 Training accuracy: 76%
 Most important features: user's average rating, number of movie ratings, genre

This tree successfully learned patterns about which users like which movies!
